# GenAI Interview Technical Assessment Preparation

Candidate Brief

Preparation Guide
● Time: 45 min
● Format: Live Screen Share
● Expected: Design + Build + Q&A
● Language Preference: Java (Python and TypeScript Welcome)
● Frameworks: ADK / LangChain4J / LangGraph

What to Expect
The session is a structured conversation, not an exam. We want to see how you think, how you work with AI tools, and how you make decisions. You'll
design a system, build a small working slice of it live (Not Expected to work), and discuss a few real-world trade-offs. Talk through your reasoning as you
go, that's more important than a perfect answer.

The Build Task

Task
Build a minimal working AI agent that takes a user question, uses at least one tool (e.g. web search, file read, or a mock knowledge base lookup), and
returns a grounded, cited answer. You choose the scope, keep it simple and working over complex and broken.
Preferred stack:
● Java 17+ LangChain4J, Google ADK (Java), Spring Boot, Any AI tool (Cursor, Claude, AntiGravity)
● You may use AI coding assistants, we want to see how well you use them
● Have your local environment ready before the session (Java, Maven/Gradle, API keys)
● A stub that runs beats a complete design that doesn't
● Be ready to explain every line you write or generate
