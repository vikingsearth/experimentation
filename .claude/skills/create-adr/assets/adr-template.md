## Notes

- Aim to keep the core decision sections to **1-2 pages**, with supplementary details linked or appended as needed
- Use **full sentences** in organized paragraphs (bullets for visual organization only)
- Focus on **rationale** (why this decision), not implementation details (how to build it)
- **One decision per ADR** - don't combine multiple decisions
- If you need extensive technical detail, link to separate design documents
- ADRs are **immutable** - if a decision changes, create a new ADR and mark the old one as "Superseded"
